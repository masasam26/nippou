# テスト仕様書 — 営業日報システム

## 1. テスト方針

### テスト種別

| 種別 | 目的 | 実施タイミング |
|---|---|---|
| 単体テスト | 個々のロジック・バリデーションの正確性確認 | 実装時・PR作成時 |
| APIテスト | エンドポイントの入出力・権限制御の確認 | PR作成時・ステージング |
| 画面テスト | UI操作・画面遷移・表示内容の確認 | ステージング |
| 結合テスト | 複数機能をまたぐシナリオの確認 | リリース前 |

### テスト環境

| 環境 | 用途 |
|---|---|
| ローカル | 開発者による単体・APIテスト |
| ステージング | QAによる画面・結合テスト |

### テストデータ共通定義

| ユーザー | role | 説明 |
|---|---|---|
| 山田 太郎 (user_id: 1) | sales | 通常の営業担当者 |
| 佐藤 花子 (user_id: 2) | sales | 別の営業担当者 |
| 鈴木 部長 (user_id: 3) | manager | 上長 |

| 顧客 | customer_id | 説明 |
|---|---|---|
| 株式会社A | 1 | 訪問記録あり |
| 株式会社B | 2 | 訪問記録あり |
| 株式会社C | 3 | 訪問記録なし |
